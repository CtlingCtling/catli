# Markdown Summary Prompt

Summarize the following markdown document concisely while preserving key information.

## Output Format

```
## Summary

<3-5 sentence summary of the main content>

## Key Points

- <bullet point 1>
- <bullet point 2>
- <bullet point 3>
- <bullet point 4>
- <bullet point 5 (if needed)>

## Important Details

<Any specific facts, numbers, or details worth noting>
```

## Guidelines

- **Summary**: Start with the document's main purpose. What problem does it solve or what topic does it cover?
- **Key Points**: Extract the most important takeaways. Limit to 5 bullets unless there's exceptional content.
- **Important Details**: Include specific values, dates, names, or decisions that might be useful for reference.
- **Tone**: Informative and neutral. Do not editorialize or add opinions.
- **Completeness**: If the document is very short (< 200 words), provide a shorter output.

## What to Include

✅ Do include:
- Main thesis or purpose
- Key concepts or techniques explained
- Important constraints or requirements
- Actionable steps or instructions
- Relevant examples

❌ Do not include:
- Generic phrases like "This document explains..." or "As discussed above..."
- Repetition of points already mentioned
- Information that contradicts or undermines the main content
